---
title: "Evaluate"
description: "POST /api/v1/evaluate"
---

Submit an action for policy evaluation. Returns a decision indicating whether the action should proceed.

```
POST /v1/evaluate
```

**Required scope:** `evaluate`

## Request body

<ParamField body="action" type="string" required>
  The tool or action name (e.g., `"process_payment"`, `"send_email"`).
</ParamField>

<ParamField body="operation" type="string">
  The operation type. One of: `read`, `write`, `create`, `delete`, `execute`. Defaults to `"execute"`.
</ParamField>

<ParamField body="target" type="object">
  Resource identifiers for the action target.
  ```json
  { "account_id": "acct_123", "resource_type": "payment" }
  ```
</ParamField>

<ParamField body="parameters" type="object" required>
  Action parameters as structured JSON.
  ```json
  { "amount": 150.00, "currency": "usd" }
  ```
</ParamField>

<ParamField body="agent_id" type="string">
  The agent submitting the action. Overrides the SDK default if set.
</ParamField>

<ParamField body="environment" type="string">
  The environment (`dev`, `stage`, `prod`). Overrides the SDK default if set.
</ParamField>

<ParamField body="sensitivity" type="array">
  Risk signal tags: `money`, `identity`, `permissions`, `regulated`, `irreversible`.
</ParamField>

<ParamField body="cost_estimate" type="object">
  Estimated cost of the action.
  ```json
  { "dollars": 1.50 }
  ```
</ParamField>

<ParamField body="idempotency_key" type="string">
  Client-generated idempotency key. Auto-generated by the SDK if not provided.
</ParamField>

## Response

<ResponseField name="decision" type="string" required>
  The policy decision: `ALLOW`, `BLOCK`, `REQUIRE_APPROVAL`, `PAUSE`, `SANDBOX`, `RATE_LIMIT`, `KILL_SWITCH`, `ESCALATE`, `ROUTE`.
</ResponseField>

<ResponseField name="allowed" type="boolean" required>
  Whether the action may proceed. `true` for `ALLOW` and `SANDBOX`, `false` for all others.
</ResponseField>

<ResponseField name="reason_codes" type="array" required>
  Machine-readable reason codes from the matched policy rule.
</ResponseField>

<ResponseField name="policy_version" type="string">
  The policy version used for evaluation (e.g., `"v2.1.0"`).
</ResponseField>

<ResponseField name="envelope_hash" type="string">
  SHA-256 hash of the evaluated envelope for integrity verification.
</ResponseField>

<ResponseField name="timing" type="object">
  Latency metadata.
  - `evaluation_ms` -- time spent evaluating the policy
  - `total_ms` -- total request time
</ResponseField>

<ResponseField name="remediation" type="object">
  Actionable guidance when a blocking rule fires. Includes `message`, `suggestion`, `type`, and `details`.
</ResponseField>

<ResponseField name="rate_limit" type="object">
  Rate limit counter snapshot (present when rate limiting is active).
  - `current` -- current count
  - `limit` -- configured limit
  - `window` -- window type
  - `resets_at` -- when the window resets
</ResponseField>

## Example

**Request:**

```bash
curl -X POST https://api.hiitl.com/v1/evaluate \
  -H "Authorization: Bearer hiitl_prod_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "action": "process_payment",
    "operation": "execute",
    "target": { "account_id": "acct_123" },
    "parameters": { "amount": 150.00, "currency": "usd" }
  }'
```

**Response (allowed):**

```json
{
  "decision": "ALLOW",
  "allowed": true,
  "reason_codes": ["STANDARD_PAYMENT"],
  "policy_version": "v2.1.0",
  "envelope_hash": "sha256:a3f2b8c9d1e4f5a6b7c8d9e0f1a2b3c4...",
  "timing": {
    "evaluation_ms": 0.8,
    "total_ms": 12.3
  }
}
```

**Response (blocked):**

```json
{
  "decision": "BLOCK",
  "allowed": false,
  "reason_codes": ["AMOUNT_EXCEEDS_LIMIT"],
  "policy_version": "v2.1.0",
  "envelope_hash": "sha256:d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9...",
  "timing": {
    "evaluation_ms": 0.6,
    "total_ms": 11.8
  },
  "remediation": {
    "message": "Payment amount exceeds the $10,000 limit.",
    "suggestion": "Reduce amount to $10,000 or below, or request approval.",
    "type": "threshold",
    "details": {
      "threshold": 10000,
      "max_allowed": 10000
    }
  }
}
```

**Response (rate limited):**

```json
{
  "decision": "RATE_LIMIT",
  "allowed": false,
  "reason_codes": ["RATE_LIMIT_EXCEEDED"],
  "policy_version": "v2.1.0",
  "rate_limit": {
    "current": 101,
    "limit": 100,
    "window": "hour",
    "resets_at": "2025-01-15T13:00:00Z"
  },
  "timing": {
    "evaluation_ms": 0.4,
    "total_ms": 10.2
  }
}
```
