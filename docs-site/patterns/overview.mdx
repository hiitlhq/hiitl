## What are patterns?

Patterns are pre-built `evaluate()` calls paired with starter policies for common AI agent actions. Each pattern includes the SDK call, a YAML policy rule, remediation guidance, and compliance notes.

Copy the pattern, customize the thresholds, and deploy.

## Pattern structure

Every pattern includes:

1. **`evaluate()` call** -- the Python SDK call to wrap the action
2. **Starter policy** -- a YAML rule with sensible defaults
3. **Remediation steps** -- what to do when a decision blocks
4. **Compliance notes** -- relevant standards (PCI-DSS, SOX, GDPR, etc.)

## Using patterns

<Steps>
  <Step title="Find the pattern">
    Browse the categories above or search for your action type.
  </Step>
  <Step title="Copy the evaluate() call">
    Paste into your tool handler. Adjust `target` and `parameters` to match your data model.
  </Step>
  <Step title="Copy the policy rule">
    Add to your `policy.yaml`. Adjust thresholds, reason codes, and metadata.
  </Step>
  <Step title="Test locally">
    Run in local mode to verify the policy fires as expected.
  </Step>
</Steps>

<Note>
All patterns use Python examples by default. The `evaluate()` call is identical in TypeScript -- see the [TypeScript SDK reference](/sdk/typescript-evaluate) for syntax.
</Note>
