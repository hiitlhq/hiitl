## Overview

Every `evaluate()` call returns a `Decision` object. It contains the evaluation outcome, timing metadata, matched rules, and guidance for what to do next.

```python
decision = hiitl.evaluate("process_payment", parameters={"amount": 500})

if decision.allowed:
    process_payment()
```

## Fields

### Core fields

<ResponseField name="decision" type="str" required>
  The decision type. One of:

  | Value | `allowed` | Meaning |
  |-------|-----------|---------|
  | `"ALLOW"` | `True` | Action is permitted |
  | `"OBSERVE"` | `True` | Action was logged but not enforced (OBSERVE mode) |
  | `"BLOCK"` | `False` | Denied by policy |
  | `"PAUSE"` | `False` | Paused pending external resolution |
  | `"REQUIRE_APPROVAL"` | `False` | Needs human review |
  | `"SANDBOX"` | `True` | Allowed in isolated/sandbox environment |
  | `"RATE_LIMIT"` | `False` | Rate limit exceeded |
  | `"KILL_SWITCH"` | `False` | Emergency stop active |
  | `"ESCALATE"` | `False` | Escalated for review |
  | `"ROUTE"` | `False` | Routed to a specific handler |
</ResponseField>

<ResponseField name="allowed" type="bool" required>
  Whether the action is permitted. `True` for `ALLOW`, `OBSERVE`, and `SANDBOX`. `False` for everything else.
</ResponseField>

<ResponseField name="reason_codes" type="list[str]" required>
  List of reason codes explaining the decision. Comes from matched policy rules (e.g., `["PAYMENT_TOO_HIGH"]`) or system defaults (e.g., `["DEFAULT_ALLOW"]`).
</ResponseField>

<ResponseField name="action_id" type="str" required>
  The auto-generated action ID for this evaluation. Format: `act_` followed by 20+ alphanumeric characters.
</ResponseField>

<ResponseField name="policy_version" type="str" required>
  Version of the policy set used for evaluation. Enables debugging and replay.
</ResponseField>

### Timing

<ResponseField name="timing" type="Timing" required>
  Performance metadata. Every decision includes timing so you can verify latency.

  | Field | Type | Description |
  |-------|------|-------------|
  | `timing.ingest_ms` | `float` | Time to build and validate the envelope |
  | `timing.evaluation_ms` | `float` | Time to evaluate the policy |
  | `timing.total_ms` | `float` | Total end-to-end time |

  ```python
  print(f"Evaluated in {decision.timing.evaluation_ms:.2f}ms")
  # Evaluated in 0.34ms
  ```
</ResponseField>

### Observe mode fields

<ResponseField name="would_be" type="str | None">
  When `decision` is `"OBSERVE"`, this shows what the decision *would have been* under enforcement. For example, `"BLOCK"` or `"REQUIRE_APPROVAL"`.
</ResponseField>

<ResponseField name="would_be_reason_codes" type="list[str] | None">
  When `decision` is `"OBSERVE"`, the reason codes that would have applied under enforcement.
</ResponseField>

### Policy details

<ResponseField name="matched_rules" type="list[MatchedRule] | None">
  Rules that matched during evaluation. Each entry has `rule_name`, `policy_set`, and `priority`.
</ResponseField>

<ResponseField name="envelope_hash" type="str | None">
  SHA-256 hash of the evaluated envelope. Allows the caller to verify that the server evaluated the exact envelope that was submitted.
</ResponseField>

### Escalation fields

<ResponseField name="resume_token" type="str | None">
  Token to correlate an escalation decision with a reviewer's response. Present when `decision` is `REQUIRE_APPROVAL`, `PAUSE`, or `ESCALATE`.
</ResponseField>

<ResponseField name="route_ref" type="str | None">
  Route configuration name from the matched rule. Used internally to resolve escalation routing.
</ResponseField>

<ResponseField name="escalation_context" type="dict | None">
  Resolved escalation routing details. Populated by the SDK when a route config is available. Contains information like webhook URLs, reviewer roles, and SLA expectations.
</ResponseField>

<ResponseField name="approval_metadata" type="ApprovalMetadata | None">
  Approval workflow details when `decision` is `REQUIRE_APPROVAL`. Contains `approval_id`, optional `sla_hours`, `reviewer_role`, and `resume_url`.
</ResponseField>

### Rate limit fields

<ResponseField name="rate_limit" type="RateLimit | None">
  Rate limit state when `decision` is `RATE_LIMIT`. Contains `scope`, `window`, `limit`, `current` count, and `reset_at` timestamp.

  ```python
  if decision.decision == "RATE_LIMIT":
      rl = decision.rate_limit
      print(f"Limit: {rl.current}/{rl.limit} in {rl.window}")
      print(f"Resets at: {rl.reset_at}")
  ```
</ResponseField>

### Sandbox fields

<ResponseField name="sandbox_metadata" type="SandboxMetadata | None">
  Sandbox routing details when `decision` is `SANDBOX`. Contains `sandbox_endpoint` and optional `sandbox_environment`.
</ResponseField>

### Error and remediation

<ResponseField name="error" type="ErrorDetail | None">
  Present when ECP itself failed (not when policy enforcement blocked an action). Contains `code` (machine-readable) and `message` (human-readable with guidance).

  ```python
  if decision.error:
      print(f"ECP error: [{decision.error.code}] {decision.error.message}")
  ```
</ResponseField>

<ResponseField name="remediation" type="Remediation | None">
  Present when a policy intentionally blocked or rate-limited an action. Contains `message`, `suggestion`, `type`, and optional `details`. Mutually exclusive with `error`.

  ```python
  if decision.remediation:
      print(f"Why: {decision.remediation.message}")
      print(f"Fix: {decision.remediation.suggestion}")
  ```
</ResponseField>

<Note>
  `error` and `remediation` are mutually exclusive. `error` means ECP failed. `remediation` means ECP worked and intentionally enforced a policy.
</Note>

## Convenience properties

These properties simplify common decision-handling patterns:

| Property | Type | Returns `True` when |
|----------|------|---------------------|
| `.ok` | `bool` | Same as `.allowed` -- action is permitted |
| `.blocked` | `bool` | Decision is `BLOCK`, `KILL_SWITCH`, or `RATE_LIMIT` |
| `.needs_approval` | `bool` | Decision is `REQUIRE_APPROVAL`, `PAUSE`, or `ESCALATE` |
| `.observed` | `bool` | Decision is `OBSERVE` (not enforced) |

```python
# These are equivalent
if decision.allowed:
if decision.ok:

# Check specific categories
if decision.blocked:
    log_blocked_action(decision.reason_codes)

if decision.needs_approval:
    queue_for_review(decision.resume_token)

if decision.observed:
    # Log what would have happened, but proceed
    log_observation(decision.would_be, decision.would_be_reason_codes)
```

## Common patterns

### Basic allow/deny

```python
decision = hiitl.evaluate("send_email", parameters={
    "to": "user@example.com",
    "subject": "Order confirmation",
})

if decision.ok:
    send_email()
else:
    handle_denial(decision.reason_codes)
```

### Full decision routing

```python
decision = hiitl.evaluate("process_payment", parameters={"amount": 5000})

match decision.decision:
    case "ALLOW":
        execute_payment()

    case "BLOCK" | "KILL_SWITCH":
        log_blocked(decision.reason_codes)
        if decision.remediation:
            notify_user(decision.remediation.suggestion)

    case "REQUIRE_APPROVAL":
        submit_for_review(
            action="process_payment",
            resume_token=decision.resume_token,
        )

    case "RATE_LIMIT":
        rl = decision.rate_limit
        retry_after = rl.reset_at if rl else None
        schedule_retry(retry_after)

    case "OBSERVE":
        # Not enforced — proceed but log
        log_observation(decision.would_be)
        execute_payment()

    case "SANDBOX":
        execute_in_sandbox(decision.sandbox_metadata)
```

### Logging decision details

```python
decision = hiitl.evaluate("delete_records", parameters={"table": "users"})

print(f"Decision: {decision.decision}")
print(f"Allowed: {decision.allowed}")
print(f"Reasons: {decision.reason_codes}")
print(f"Policy: {decision.policy_version}")
print(f"Latency: {decision.timing.total_ms:.2f}ms")

if decision.matched_rules:
    for rule in decision.matched_rules:
        print(f"  Matched: {rule.rule_name} (priority {rule.priority})")
```
